# Agent Hub — 界面与交互说明（供 UI 设计）

> **用途**：供 UI/UX 设计师在**不读代码**的前提下理解当前产品界面结构、用户路径与交互约束，便于重新设计视觉与布局。  
> **产品**：本地桌面客户端（Tauri）+ Web 前端（React），工程路径：`Apps/agent-hub/`。  
> **语言**：当前界面文案以**简体中文**为主。

---

## 1. 产品概述

- **定位**：用户选择一位「助手」（Agent），与其多轮对话；对话由远端大模型（当前为 **DeepSeek**，OpenAI 兼容接口）驱动。
- **本地数据**：助手列表、人设与能力绑定、会话与消息等存于本机 SQLite；API Key 存系统凭据（keyring），不写入业务数据库明文。
- **能力（Tools）**：模型可调用内置工具（如诚实声明、事务委派、天气查询等）；用户为每位助手勾选可选能力，必选能力不可关闭。

---

## 2. 平台与实现约束（设计时需知情）

| 项目 | 说明 |
|------|------|
| 运行形态 | **桌面窗口**（Tauri 内嵌 WebView），非纯网页；设计需考虑窗口缩放、最小尺寸。 |
| 视口高度 | 主壳使用 `100dvh` 级全屏高度，避免移动端浏览器地址栏问题；桌面端以**可调整窗口**为主。 |
| 技术栈（实现侧） | React、Tailwind CSS、Radix UI 风格组件；主题通过 CSS 变量（`hsl(var(--primary))` 等），**可整体换肤**。 |
| 当前反馈方式 | 成功/失败大量使用 `window.alert` 与 `window.confirm`，**非** Toast 体系；重设计时可规划统一反馈组件，实现需另排期。 |

## 4. 主界面布局

### 4.1 顶栏（Header）

- **左侧**：应用图标（Bot）+ 文案「智能助手」。
- **右侧**：「大模型密钥」图标按钮（钥匙图标）。
  - **状态点**：右上角小圆点，**绿色**表示当前检测到大模型密钥已配置；**灰色**表示未配置或不可用。
- **交互**：点击打开「大模型密钥」弹窗（见 §6.1）。

### 4.2 左侧栏（Aside，约 260px 宽）

- **区域标题**：「对话对象」（小写宽字样式）。
- **助手列表**：
  - 加载中：骨架屏（多条占位条）。
  - 空列表：说明文案「暂无助手。点击下方「助手与数据」创建。」
  - 有数据：每条为**可选中按钮**，显示 `display_name`；选中态为强调边框/背景。
- **底部分隔线** + **「助手与数据」** 按钮（齿轮图标），**全宽** 次要按钮样式。
- **交互**：点击助手 → 切换当前对话对象；点击「助手与数据」→ 打开助手管理弹窗（见 §6.2）。

### 4.3 主内容区（Main）

#### A. 未选择助手且未加载中

- 居中 **空态卡片**：标题「开始对话」，说明「请选择一位助手以开始对话。」

#### B. 已选择助手

整体为一张 **大卡片**，分三块：

1. **卡片标题栏**
   - 左侧：当前助手名称（随切换动画）。
   - 右侧：**大模型** 下拉框（`Label` + `select`），当前仅 **DeepSeek** 一项；会话存在时可切换（若未来多模型会扩展选项）。
   - 若未配置密钥：标题栏下方一行提示「请先在右上角配置当前大模型对应的密钥后再发送。」

2. **密钥缺失提示条**（未配置时）
   - 琥珀色背景条，说明需点击右上角保存密钥（与 DeepSeek OpenAI 兼容接口一致）。

3. **消息区**
   - 可滚动列表；用户消息与助手消息**左右对齐**（用户偏右、助手偏左），不同背景样式。
   - 加载会话中：骨架屏占位。
   - 无消息：居中「还没有消息。」
   - 发送中：底部显示**阶段文案**（见 §5.2）+ 旋转图标。

4. **输入区**
   - 多行文本框（`Textarea`），占位「输入消息…（Enter 发送，Shift+Enter 换行）」。
   - **Enter** 发送，**Shift+Enter** 换行。
   - 右侧发送按钮（纸飞机图标）；发送中显示加载图标并禁用。

## 5. 对话与实时反馈

### 5.1 发送前置条件

- 若当前大模型预设未配置密钥：**阻止发送**，并弹窗提示需先配置密钥。

### 5.2 回复阶段（Thinking 状态）

- 应用监听后端事件 `chat_reply_phase`，载荷含 `conversation_id` 与 `phase`。
- **阶段 → 界面文案**（可随设计改为更细的进度条或骨架）：

| `phase` | 展示文案 |
|---------|----------|
| `thinking` | 正在思考… |
| `analyzing` | 正在分析上下文… |
| `tools` | 正在调用工具… |
| `generating` | 正在生成回复… |

- 未知 `phase` 时回退为「正在处理…」。

### 5.3 发送后

- 成功后刷新消息列表（乐观更新可能先插入用户消息）。
- 失败时 `alert` 错误，并尝试恢复消息列表。

## 6. 弹窗与流程

### 6.1 「大模型密钥」弹窗

- **标题**：大模型密钥  
- **说明**：API Key 保存在本机；同一模型新密钥覆盖旧密钥。
- **表单**：
  - 大模型下拉（当前仅 DeepSeek）。
  - 密码型输入框「API 密钥」。
- **操作**：
  - **清除**：清除该大模型已保存的密钥。
  - **保存**：保存密钥；关闭弹窗；刷新密钥状态点。
- **关闭**：点击遮罩或关闭，会刷新密钥检测。

### 6.2 「助手与数据」弹窗（核心）

**设计原则（产品侧）**：说明与设置尽量**以「能力」为单位**组织：助手标识、对话策略（System）、各 Tool 能力卡片 + 委派策略等。

#### 结构概览

1. **标题**：助手与数据  
2. **工具条**：创建（+）、导入 JSON（上传图标）  
3. **已有助手**（有数据时）：短列表，点击加载该助手到表单  
4. **可滚动区域**（有最大高度，内容多时可纵向滚动）：
   - **助手标识**卡片：名称输入（说明与能力区分）
   - **能力配置**标题 + 一段总说明
   - **对话策略**卡片：System Prompt 多行输入，带「System」标签
   - **能力目录**：按能力分卡片，每条来自后端 `skills_catalog`（名称、描述、必选/可选）
     - **委派能力**（`request_agent_help`）：卡片内嵌「委派策略」——出站委派开关、入站委派开关及说明
     - **诚实声明**等必选：说明「已启用，不可关闭」
     - **可选能力**（如天气）：「启用此能力」开关
5. **底栏**：
   - 编辑模式：删除助手（危险操作）  
   - 创建模式：「快速创建」（默认模板助手）  
   - 关闭、保存/创建（对勾）

#### 关键交互

| 操作 | 行为 |
|------|------|
| 打开弹窗 | 拉取能力目录；新建模式默认勾选必选能力 |
| 点击已有助手 | 并行加载能力目录与助手详情（含 `skill_ids`） |
| 快速创建 | 后端创建默认模板助手，刷新列表并关闭弹窗 |
| 导入 JSON | 选择文件，数组批量导入助手；`alert` 导入数量 |
| 保存/创建 | 校验名称非空；提交 `skillIds` 等；刷新列表；关闭弹窗 |
| 删除 | `confirm` 确认后删除 |

#### 校验与约束

- 名称必填；空人设会有后端默认文案。
- 必选能力 ID 由后端合并，前端不可关闭对应项。
- 弹窗内容区需支持**小屏/矮窗口**下仍可滚动，避免能力区不可见。

## 7. 数据与状态（供设计稿标注）

### 7.1 助手（列表项）

- 展示：`display_name`  
- 不直接在列表展示：委派开关、能力列表（在详情弹窗内）

### 7.2 能力卡片（动态）

- 字段：`tool_name`（标题）、`description`（说明）、`is_required`（必选/可选徽标）
- 可选能力：依赖用户是否勾选「启用此能力」

### 7.3 会话与消息

- 消息角色：`user` | `assistant`（及未来可能扩展）
- 每条为纯文本 `content`（当前无富媒体/附件）

---

## 8. 无障碍与键盘

- 主要输入框、按钮宜保留可聚焦样式（focus ring）。
- 对话输入框：Enter 发送、Shift+Enter 换行（需在设计上避免与可访问性冲突，如可注明「与常见 IM 一致」）。
- 图标按钮需提供 `aria-label` / `title`（实现侧已有部分）。

---

## 9. 视觉与品牌（当前实现）

- 背景：浅色 + 顶部径向渐变弱光。
- 主色：通过 `--primary` 等变量控制，**未绑定独立品牌手册**；重设计时可整套替换色板与圆角体系。
- 圆角：卡片、按钮、输入框统一为中等圆角（Tailwind 默认尺度）。

## 10. 设计交付建议清单

设计师可优先交付：

1. **桌面端布局栅格**（侧栏宽度、主区最小宽度、弹窗最大宽高）。
2. **设计系统**：主/辅色、语义色（成功/警告/危险）、字号阶梯、间距、圆角、阴影。
3. **组件规范**：按钮层级、列表项选中态、卡片、开关、弹窗、表单、空态/加载态。
4. **助手与数据弹窗**线框图 + 能力卡片的变体（必选/可选/含委派策略）。
5. **对话区**气泡与输入区布局（含「回复中」状态）。
6. **密钥状态**在顶栏的表达方式（点/文案/图标）。
