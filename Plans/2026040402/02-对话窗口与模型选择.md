# 02 — 对话窗口：可选大模型（首期 DeepSeek）

## 用户问题

在 **对话窗口** 增加 **可选大模型**能力；当前产品 **仅提供 DeepSeek** 作为可选后端（下拉中可出现更多项时再扩展，**不**改「会话内自选模型」这一交互形态）。

## 问题解析

- **协议**：DeepSeek 提供 **OpenAI 兼容 HTTP API**，与父批次 **OpenAI 兼容 + async-openai + 可配 `base_url`** 一致，**不**引入本地模型。
- **密钥**：API Key 仍只经由 **Rust Hub + keyring**（或既有全局密钥模型）；**禁止**在渲染进程拼接 Key。
- **粒度**：建议 **按会话（conversation）** 记录当前选用的预设；**同一助手**可在不同会话中使用不同模型（若产品希望「助手级默认」可再加一层默认值，与会话值合并：会话覆盖助手默认）。

## 解答或实现方案

### 1. UI（对话窗口）

- 在对话窗口 **工具栏 / 顶栏** 提供 **大模型** 控件：**下拉或分段控件**。
- **首期**：下拉中 **仅一项可选：DeepSeek**（或仅 DeepSeek 可点选，其它若占位则禁用），仍保留「可选择」的交互位置，便于后续追加厂商而少改布局。
- **可选**：旁注短说明——「影响 **本会话** 使用的远程模型（DeepSeek 兼容接口）」。

### 2. 预设定义（首期）

| 预设 id（实现锁版） | 展示名 | `base_url` | 模型 id（示例，以 DeepSeek 官方为准） |
|---------------------|--------|------------|----------------------------------------|
| `deepseek_default`  | DeepSeek | `https://api.deepseek.com`（**路径 / v1 后缀以官方 OpenAI 兼容文档为准**） | 例：`deepseek-chat` — **实现前核对官方表** |

- Hub 出站请求使用：**会话所选预设** 的 `base_url` + **模型名** + keyring 中 **DeepSeek**（或「兼容端点」）密钥条目；**具体 env/配置键名在 `Apps/` 定稿**。

### 3. 数据与路由

- 表 **`conversations`**（或父批 §3.4 落库时的等价结构）增加字段，例如 **`llm_preset_id: string`**（或 `provider` + `model_id` 两列）；**新建会话** 时写入当前 UI 选择，**切换模型** 更新当前会话并 **对后续轮次生效**（已生成历史不回溯改写）。
- 若未配置 Key：控件可 **禁用** 或点选后提示 **先去设置绑定 DeepSeek Key**（文案产品定稿）。

### 4. 与助手编辑的关系

- **正交**：助手人设 / Skills / 入站委派等 **不** 因本下拉而变；仅 **本会话 LLM 路由** 变化。
- 可选后续：在 **助手编辑** 增加「默认大模型预设」— **非本期必用**；本期 **对话窗选择** 即可闭环 M2。

### 5. 里程碑

- 与父批次 **M2** 一并验收：**能对话** + **窗口内可选 DeepSeek（首期唯一选项）** + 会话级落库。
