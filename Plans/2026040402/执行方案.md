## 1. 用户问题（本批次）

在父批次架构下：安装完成、初次打开并配置 Key 后，用户 **无需先新建** 即可与 **已存在的一条助手** 对话；该条为 **默认模板** 生成结果（**仅必要 Skills**）。**首装不做任何专门针对「首条」的运行时特殊化**。**新建助手** 须有 **「默认模板」** 按钮，与首装 **同一路径**。**对话窗口** 须支持 **自选大模型**，当前 **仅 DeepSeek**（见 **02**）。

## 2. 问题解析（本批次）

- **范围**：全局 `skills`、`必要 Skills` 清单、首装 **= 一次默认模板调用**、新建流程 **「默认模板」** 按钮；**会话级 LLM 预设**（首期 DeepSeek，见 **02**）。
- **依赖**：父批次 §3.3～3.5；Agent **§3.1**；LLM **§2**（OpenAI 兼容）；表 **§3.4**（`conversations` 扩展字段）。

## 3. 解答 / 可执行结论

详见 **[01-默认Agent与内置委派Skill.md](./01-默认Agent与内置委派Skill.md)**。要点：

1. **业务代码**：**禁止** 以「是否首装那条 / 固定 UUID」分支 Hub、委派、配额、审计（`**allows_outgoing_delegation` / `accepts_incoming_delegation` 对所有 Agent 同一语义**）。
2. **首装**：空库 / 无 Agent 条件下 `**app_meta.first_default_template_applied` 未执行时** **调用默认模板核心一次**（**方案 B**，幂等与 meta 见 **01 §2**）；**无**首装专属后续逻辑；**迁移不插**首条 `agents`。
3. **删光后再要同款**：**新建 → 默认模板** → **新 `agent_id`**，与首装 **同一实现**。
4. **必要 Skills**：现时至少委派 + 规则类；清单可扩展；首装与默认模板 **仅绑清单内 Skill**，无多余 Skill。**规则类 builtin MVP** = **纯提示档**（tool 元数据 + `system_prompt` + **轻 handler**；不做语义级捏造判定、不单开与委派同级的大型审计表）；见 **01 · §3**。
5. **编辑助手**：**出站** — **「如需其他助手辅助时是否允许委派」** → `**allows_outgoing_delegation`**；**入站** — **「是否可以被其他助手委派事务」** → `**accepts_incoming_delegation`**（均见父批 §3.1）。**MVP** 委派许可由二者与 Hub 配额等 **共同判定**，**不以 `delegate_allowlist` 为门禁**。UI、Hub 见 **01 · §5**（§5.1 / §5.2）。
6. **对话窗口**：**大模型** 下拉，首期仅 **DeepSeek**；会话存 `**llm_preset_id`**（或等价）；Hub 按预设走 `base_url` / 模型名；见 **02**。

## 5. 验收标准（本批增量）

1. 首装空库：有一条可对话 Agent；Key 配置流程后可用。
2. 首装 Agent **仅** 绑定 **必要 Skills**（现时委派 + 规则类）；**不**向用户暴露「首装特殊身份」；id 展示策略与普通 Agent 一致。**规则类** 首版为 **纯提示** + 轻 handler，与 **01 §3** 一致。
3. **可删除** 该 Agent，行为与删除任意 Agent 一致。
4. **「默认模板」** 产生 **新 `agent_id`**，实现与首装 **同源**；**禁止** 仅首装可用的隐藏 API。
5. 代码审查：**无** `is_default` / 魔法 UUID **参与** Hub 决策的路径。
6. **编辑助手** 含 **出站 + 入站** 两委派开关（**01 §5.1 / §5.2**）：主叫 **关出站** 时 Hub **禁止**其发起委派；被叫 **关入站** 时 **前置拒绝**入站且 **不参与** 目标发现。**入站开** 时 **积极** 含 Hub `**T_accept`** 内 `**DelegateResponse**`，**超时视为不积极**（见 **01 §5.2**）。
7. **对话窗口** 大模型选择：**DeepSeek** 可走通；切换写入会话字段，与 **02** 一致。
